# Percorso di certificazione per Adobe [!DNL Workfront Fusion] Developer Professional

{{intro}}

>[!NOTE]
>
>**Nuove risorse di studio disponibili:** La [Guida alla preparazione degli esami](https://app.rockinfo.com/courses/228) fornisce un percorso personalizzato attraverso il processo di preparazione degli esami di certificazione, inclusi i materiali necessari per ottenere questa certificazione. _Se viene richiesto di accedere alla Guida alla preparazione esami, utilizzare il proprio Adobe ID._

## Descrizione esame {#exam-description}

{{exam-description}}

<br>

### Dettagli esame {#exam-details}

* Livello: professionale (0-12 mesi di esperienza)
* Punteggio minimo: 33/51
* Tempo: 102 minuti
* Distribuzione: online protetta (richiede l&#39;accesso alla fotocamera)
* Lingue disponibili: inglese
* Costo: $125 (globale) / $95 (India)
* ID esame: AD0-E902

<br>

### Pubblico previsto {#intended-audience}

* Sviluppatore, ingegnere di automazione
* Automator, Process Optimization e Automation Manager
* Consulenti per l&#39;integrazione, consulenti, partner
* Integratori di sistemi/partner
* Analisti aziendali, analisti aziendali applicati
* Amministratore di sistema (IT, MIS, tecnologia, Knowledge Systems, amministratori di IA)

<br>

### Esperienza minima {#minimum-experience}

Dovresti avere:

* Capacità di utilizzare l&#39;API [!DNL Workfront] e l&#39;API Explorer, ma non necessariamente in modo efficiente nelle chiamate API
* Possibilità di utilizzare chiamate API personalizzate per l&#39;utilizzo di [!DNL Workfront] e altre API
* Possibilità di utilizzare i moduli per l&#39;app [!DNL Workfront] e altre azioni quali creazione, lettura, aggiornamento, eliminazione e ricerca
* La capacità di creare soluzioni valide che producono risultati aziendali senza colpire i guardrail [!DNL Fusion]
* Possibilità di selezionare un modulo di attivazione appropriato per ottenere risultati aziendali
* Informazioni sull&#39;utilizzo del controllo di flusso (route, iteratori/aggregatori, explain flow)
* Possibilità di eseguire operazioni di base per la risoluzione dei problemi e la gestione degli errori
* La capacità di testare in modo efficace e andare &quot;live&quot; con uno scenario
* Possibilità di gestire team e utenti dell’organizzazione
* Informazioni sullo scopo e sull’utilizzo dei connettori universali per l’API REST Oauth2

<br>

### Obiettivi e ambito dell&#39;esame {#exam-objectives}

{{exam-objectives}}

**Sezione 1: concetti tecnici fondamentali (39%)**

* Dato uno scenario [!DNL Fusion] con un formato di campo che deve essere in un altro formato di campo, fornisci le possibili funzioni che potrebbero essere utilizzate per trasformare correttamente i dati
* Dato uno scenario [!DNL Fusion] che richiede che una funzione sia nidificata all&#39;interno di un&#39;altra funzione, selezionare l&#39;espressione formattata correttamente
* Identificare i modi in cui utilizzare i team per controllare l&#39;accesso alle connessioni e ad altre funzionalità di [!DNL Fusion]
* Dato uno scenario [!DNL Fusion], identificare le limitazioni di sistema e funzionalità
* Dato uno scenario [!DNL Fusion] che richiede la manipolazione del fuso orario, distinguere tra utente, organizzazione visibile e dove è necessario ignorare il fuso orario
* Dato uno scenario [!DNL Fusion], selezionare i moduli o le azioni [!DNL Workfront] appropriati
* Dato uno scenario [!DNL Fusion], seleziona l’espressione del pannello di mappatura e/o i moduli che trasformerebbero in modo appropriato i dati di input in dati di output
* Identificare il modo corretto di utilizzare lo strumento di sviluppo [!DNL Fusion] per risolvere gli errori di esecuzione o determinare le chiamate e le risposte ai sistemi di terze parti
* Dato uno scenario [!DNL Fusion] in cui i dati in un percorso sono necessari in un altro percorso, fornire una soluzione set-get per rendere tali dati disponibili su percorsi di routing aggiuntivi
* Identificare i modi in cui utilizzare o produrre un modello [!DNL Fusion] per un&#39;esigenza aziendale
* Dato uno scenario [!DNL Fusion], identifica cosa fare per visualizzare esecuzioni e/o risolvere errori
* Identificare le diverse opzioni nelle impostazioni dello scenario [!DNL Fusion] per soddisfare le esigenze dello scenario
* Dato uno scenario [!DNL Fusion], selezionare il controllo di flusso appropriato e/o determinare la combinazione corretta dei moduli di controllo di flusso

**Sezione 2: Progettazione e architettura dello scenario (35%)**

* Dato un elenco di passaggi, scegli la sequenza corretta per analizzare JSON e convertirlo in bundle
* Dato un set di requisiti, identifica il metodo di ricerca dei dati corretto
* Identifica le differenze tra i trigger istantanei e pianificati
* Identificare un modulo corretto per caricare un documento in [!DNL Workfront]
* Dato un requisito aziendale, identificare l&#39;approccio corretto per tenere traccia del record elaborato da [!DNL Fusion] (non da un utente) e quando
* Dato uno scenario [!DNL Fusion], identifica un&#39;opportunità per semplificare la progettazione e/o ottimizzare per la manutenzione
* Identifica [!DNL Fusion] best practice per la documentazione degli scenari
* Data una situazione, identifica un’opportunità per ridurre il flusso di dati attraverso uno scenario
* Dato un esempio del numero di bundle visualizzati in un modulo di ricerca e in un aggregatore seguente, spiegare le modifiche apportate al numero di operazioni elaborate e visualizzate dalla finestra di ispezione del bundle
* Identificare le informazioni presenti nella finestra di ispezione del bundle
* Data una descrizione della necessità di archiviare uno scenario per più di 60 giorni e selezionare l’unico modo possibile (blueprint di importazione/esportazione)
* Dato un esempio di errore 403: proibito, seleziona correttamente l’origine del messaggio di errore e causa

**Sezione 3: test e gestione degli errori (16%)**

* Identificare gli elementi di un piano di test
* Identificare gli elementi di un test case
* Dato uno scenario [!DNL Fusion], identificare correttamente la direttiva appropriata per gestire un servizio inaffidabile
* Identificare il processo per aggiungere la gestione degli errori personalizzata in uno scenario [!DNL Fusion]
* Identificare come tenere traccia, leggere e risolvere le esecuzioni incomplete
* Dato uno scenario [!DNL Fusion] con dati richiesti mancanti, seleziona i modi per gestire i dati non validi

**Sezione 4: Utilizzo delle API (10%)**

* Dato un API di terze parti che restituisce un errore 429: Troppe richieste, identifica una soluzione per evitare o gestire l’errore
* Dato uno scenario [!DNL Fusion] in cui la nuova funzionalità non è disponibile nel modulo [!DNL Workfront], identificare il riferimento e il tipo di modulo corretti per utilizzare la nuova funzionalità
* Dato un sistema di terze parti privo di un’app dedicata, identifica l’app HTTP e seleziona il modulo appropriato
* Identifica la differenza tra le operazioni CRUD e altre funzionalità comuni delle API REST

## Descrizione esame {#exam-description}

{{exam-description}}

<br>

### Dettagli esame {#exam-details}

* Livello: professionale (0-12 mesi di esperienza)
* Punteggio minimo: 33/51
* Tempo: 102 minuti
* Distribuzione: online protetta (richiede l&#39;accesso alla fotocamera)
* Lingue disponibili: inglese
* Costo: $125 (globale) / $95 (India)
* ID esame: AD0-E902

<br>

### Pubblico previsto {#intended-audience}

* Sviluppatore, ingegnere di automazione
* Automator, Process Optimization e Automation Manager
* Consulenti per l&#39;integrazione, consulenti, partner
* Integratori di sistemi/partner
* Analisti aziendali, analisti aziendali applicati
* Amministratore di sistema (IT, MIS, tecnologia, Knowledge Systems, amministratori di IA)

<br>

### Esperienza minima {#minimum-experience}

Dovresti avere:

* La capacità di con [!DNL Workfront] API ed API Explorer, ma non necessariamente esperti nelle chiamate API
* Possibilità di utilizzare chiamate API personalizzate per l’utilizzo di [!DNL Workfront] e altre API
* Possibilità di utilizzare i moduli per [!DNL Workfront] app e altre azioni come creare, leggere, aggiornare, eliminare e cercare
* La capacità di creare soluzioni valide che producano risultati aziendali senza colpire [!DNL Fusion] guardrail
* Possibilità di selezionare un modulo di attivazione appropriato per ottenere risultati aziendali
* Informazioni sull&#39;utilizzo del controllo di flusso (route, iteratori/aggregatori, explain flow)
* Possibilità di eseguire operazioni di base per la risoluzione dei problemi e la gestione degli errori
* La capacità di testare in modo efficace e andare &quot;live&quot; con uno scenario
* Possibilità di gestire team e utenti dell’organizzazione
* Informazioni sullo scopo e sull’utilizzo dei connettori universali per l’API REST Oauth2

<br>

### Obiettivi e ambito dell&#39;esame {#exam-objectives}

{{exam-objectives}}

**Sezione 1: concetti tecnici fondamentali (39%)**

* Dato un [!DNL Fusion] con un formato di campo che deve essere in un altro formato, fornisci le possibili funzioni che possono essere utilizzate per trasformare correttamente i dati
* Dato un [!DNL Fusion] scenario che richiede che una funzione sia nidificata all&#39;interno di un&#39;altra funzione, seleziona l&#39;espressione formattata correttamente
* Identificare i modi in cui utilizzare i team per controllare l&#39;accesso a connessioni e altro [!DNL Fusion] funzioni
* Dato un [!DNL Fusion] identificare le limitazioni di sistema e funzionalità
* Dato un [!DNL Fusion] scenario che richiede la manipolazione del fuso orario, distinguere tra utente, organizzazione visibile e dove è necessario ignorare il fuso orario
* Dato un [!DNL Fusion] scenario, seleziona l’opzione appropriata [!DNL Workfront] moduli o azioni
* Dato un [!DNL Fusion] , seleziona l’espressione e/o i moduli del pannello di mappatura che trasformerebbero in modo appropriato i dati di input in dati di output
* Identificare il modo o i modi corretti per utilizzare [!DNL Fusion] Strumento di sviluppo per risolvere gli errori di esecuzione o determinare le chiamate e le risposte effettuate a sistemi di terze parti
* Dato un [!DNL Fusion] scenario in cui i dati in un percorso sono necessari in un altro percorso, fornire una soluzione set-get per rendere tali dati disponibili su percorsi di routing aggiuntivi
* Identificare i modi in cui utilizzare o produrre un [!DNL Fusion] modello per esigenze aziendali
* Dato un [!DNL Fusion] scenario, identifica cosa fare per visualizzare le esecuzioni e/o risolvere gli errori
* Identificare le diverse opzioni in [!DNL Fusion] impostazioni dello scenario per soddisfare le esigenze dello scenario
* Dato un [!DNL Fusion] scenario, selezionare il controllo di flusso appropriato e/o determinare la combinazione corretta dei moduli di controllo di flusso

**Sezione 2: Progettazione e architettura dello scenario (35%)**

* Dato un elenco di passaggi, scegli la sequenza corretta per analizzare JSON e convertirlo in bundle
* Dato un set di requisiti, identifica il metodo di ricerca dei dati corretto
* Identifica le differenze tra i trigger istantanei e pianificati
* Identificare un modulo corretto per il caricamento di un documento in [!DNL Workfront]
* Dato un requisito aziendale, identifica l’approccio corretto per tenere traccia di tale [!DNL Fusion] ha elaborato un record (non un utente) e quando
* Dato un [!DNL Fusion] identificare un&#39;opportunità per semplificare la progettazione e/o ottimizzare la manutenzione
* Identificare [!DNL Fusion] best practice per documentare gli scenari
* Data una situazione, identifica un’opportunità per ridurre il flusso di dati attraverso uno scenario
* Dato un esempio del numero di bundle visualizzati in un modulo di ricerca e in un aggregatore seguente, spiegare le modifiche apportate al numero di operazioni elaborate e visualizzate dalla finestra di ispezione del bundle
* Identificare le informazioni presenti nella finestra di ispezione del bundle
* Data una descrizione della necessità di archiviare uno scenario per più di 60 giorni e selezionare l’unico modo possibile (blueprint di importazione/esportazione)
* Dato un esempio di errore 403: proibito, seleziona correttamente l’origine del messaggio di errore e causa

**Sezione 3: Test e gestione degli errori (16%)**

* Identificare gli elementi di un piano di test
* Identificare gli elementi di un test case
* Dato un [!DNL Fusion] individuare correttamente la direttiva appropriata per gestire un servizio inaffidabile
* Identifica il processo per aggiungere la gestione personalizzata degli errori in un [!DNL Fusion] scenario
* Identificare come tenere traccia, leggere e risolvere le esecuzioni incomplete
* Dato un [!DNL Fusion] scenario con dati richiesti mancanti, seleziona modi per gestire i dati non validi

**Sezione 4: Utilizzo delle API (10%)**

* Dato un API di terze parti che restituisce un errore 429: Troppe richieste, identifica una soluzione per evitare o gestire l’errore
* Dato un [!DNL Fusion] scenario in cui non è disponibile una nuova funzionalità nel [!DNL Workfront] identificare il riferimento e il tipo di modulo corretti per utilizzare la nuova funzionalità
* Dato un sistema di terze parti privo di un’app dedicata, identifica l’app HTTP e seleziona il modulo appropriato
* Identifica la differenza tra le operazioni CRUD e altre funzionalità comuni delle API REST

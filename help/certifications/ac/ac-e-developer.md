# Percorso di certificazione per Adobe [!DNL Commerce] Sviluppatore esperto

Scopri come diventare un esperto Adobe Developer certificato.

## Esperienza minima

Dovresti avere almeno 1-3 anni di esperienza con l&#39;Adobe [!DNL Commerce] e conosci le tecnologie e gli ambienti seguenti:

* Adobe [!DNL Commerce] - versione 2.4
* PHP
* Strumenti di debug XDebug/PHP, strumenti di debug del browser
* Strumenti di stampa
* Compositore
* SSH, Linux di base
* Redis, Elasticsearch, SQL, Varnish, RabbitMQ
* Strumenti di convalida API, strumenti client API (Postman)
* Software di controllo delle versioni come Git
* Architettura cloud

>[!NOTE]
>
>La tua certificazione è valida per due anni dal momento del passaggio. È consigliabile adottare la versione più recente quando possibile, in quanto copre le funzionalità di prodotto più recenti.

**Obiettivi e campo di applicazione dell&#39;esame**

Sezione 1: Adobe [!DNL Commerce] Tecniche di architettura e personalizzazione (36%)

* Dimostrare come utilizzare efficacemente la cache in Adobe [!DNL Commerce]
* Creare, utilizzare e manipolare attributi di estensione personalizzati
* Consiglia soluzioni per come applicare gli aggiornamenti personalizzati di tema alle pagine di prodotti/categorie
* Applicare correttamente osservatori, preferenze e plug-in (effetti di ordinamento)
* Dimostrare la capacità di utilizzare il livello di configurazione in Adobe [!DNL Commerce]
* Dimostrare la conoscenza del funzionamento dei percorsi nell&#39;Adobe [!DNL Commerce]
* Dimostrare la capacità di personalizzare pagebuilder
* Determinare gli effetti e i vincoli della configurazione di più siti in una singola istanza
* Descrizione delle funzionalità e dei vincoli dell&#39;iniezione di dipendenza
* Descrizione di come aggiungere e configurare i campi nelle impostazioni dell&#39;archivio
* Spiegare i casi d’uso per le patch Git e le modifiche a livello di file in Composer
* Creare nuovi comandi in CLI
* Dimostrare come scrivere un test di integrazione
* Identifica Adobe [!DNL Commerce] funzionalità di sicurezza (CSP, escape, chiavi modulo, pulizia, reCAPTCHA, convalida dell&#39;input)
* Spiegare come funziona il sistema di programmazione CRON
* Dimostrare la capacità di caricare e manipolare i dati
* Dimostrare la capacità di utilizzare l’emulazione dell’app

Sezione 2: Utilizzo dei database e dell&#39;EAV (10%)

* Manipolare attributi EAV e set di attributi a livello di programmazione
* Dimostrare la capacità di estendere lo schema del database
* Dimostrare la capacità di importare/esportare dati da Adobe Commerce
* Descrivere come utilizzare le patch e le impostazioni ricorrenti per modificare il database

Sezione 3: Sviluppo con l’amministratore (6%)

* Dimostrare la capacità di aggiornare e creare griglie e moduli
* Estendere le azioni Griglia
* Dimostrare la possibilità di creare classi modificatore
* Dimostrare la capacità di limitare l&#39;accesso all&#39;ACL

Sezione 4: Personalizzazione del catalogo (6%)

* Applicare modifiche ai tipi di prodotto esistenti e crearne di nuovi
* Modificare ed estendere le entità Catalogo esistenti
* Dimostrare la capacità di gestire gli indici e personalizzare la produzione dei prezzi
* Spiegare in che modo l’inventario multi-sorgente influisce sulle scorte (a livello di programma)

Sezione 5: Personalizzazione delle operazioni di vendita (6%)

* Dimostrare la capacità di sviluppare nuovi metodi di pagamento o personalizzare i metodi di pagamento esistenti
* Dimostrare la possibilità di aggiungere e personalizzare i metodi di spedizione
* Dimostrare la capacità di personalizzare le operazioni di vendita
* Spiegare come personalizzare i totali

Sezione 6: API e servizi (4%)

* Dimostrare la possibilità di creare nuove API o estendere le API esistenti
* Dimostrare la capacità di utilizzare il sistema di accodamento

Sezione 7: Adobe [!DNL Commerce] Architettura cloud (16%)

* Dimostrare la conoscenza dell&#39;Adobe [!DNL Commerce] flusso di lavoro architettura/ambiente
* Dimostrare di avere una conoscenza operativa dei file di progetto cloud, delle autorizzazioni e della struttura
* Dimostrare la capacità di configurare archivi basati su più domini su Adobe [!DNL Commerce] Cloud (siti con più domini, funzioni di sviluppo e supporto)
* Dimostrare una conoscenza generale dei servizi applicativi e delle modalità di gestione (YAML, PHP, MariaDB, Redis, RabbitMQ, ecc.)
* Identificare come accedere a diversi tipi di log
* Dimostrare la capacità di implementare un progetto (passaggi principali della distribuzione)
* Definisce le caratteristiche fornite dagli strumenti ECE
* Identificare gli usi per le patch ECE (violazione della sicurezza)
* Descrivere come mantenere e aggiornare gli strumenti ECE
* Distinguere quando contattare il supporto *File e limitazioni Yaml (biglietti DIY vs Support)
* Dimostrare la conoscenza di base delle funzioni di configurazione e installazione di OOTB FASTLY

Sezione 8: Configurazione/Adobe [!DNL Commerce] Cloud (10%)

* Descrivi come impostare/configurare un Adobe [!DNL Commerce] Cloud
* Applica le informazioni di base sulla risoluzione dei problemi di Cloud (gerarchia dell’interfaccia utente web e variabili, precedenza delle configurazioni)
* Dimostrare la comprensione della gestione degli utenti cloud e dell’interfaccia utente di onboarding
* Descrizione di come aggiornare le variabili cloud utilizzando l’interfaccia utente
* Descrizione della gestione dell’ambiente tramite l’interfaccia utente
* Dimostrare la comprensione del branching tramite l’interfaccia utente
* Identificare le funzionalità di Adobe Commerce Cloud Plan

Sezione 9: [!DNL Commerce] Strumento Cloud CLI (gestione) (6%)

* Dimostrare di comprendere l’aggiornamento delle variabili cloud tramite CLI
* Dimostrare la comprensione dell’ambiente Management utilizzando CLI (funzionalità esclusive di CLI: attivazione di e-mail, ambienti di rebase, snapshot, dump di db, configurazione dell’ambiente locale)
* Dimostrare la comprensione del ramo utilizzando CLI
* Dimostrare come risolvere i problemi relativi ai servizi cloud? (My SQL, Redis, tunnel:info)

**Fai un test pratico**

Prima di iniziare, consigliamo di effettuare il test gratuito. Avrai accesso illimitato a tutti i test di esercitazione disponibili per gli esami a livello di esperti di Adobe.

I nostri test sono tutti sviluppati partendo dallo stesso modello degli esami in diretta. Il test di esercitazione ti aiuterà a comprendere gli argomenti trattati nell&#39;esame in diretta.

Ricordare che il superamento di un test di pratica non garantisce il superamento dell&#39;esame di certificazione.

Vai a [Portale di preparazione all&#39;esame di certificazione di Adobe](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} ora per fare un test di allenamento.

**Prendere l&#39;esame di certificazione**

Adobe utilizza due fornitori per amministrare gli esami elaborati, PSI ed Examity.

* **PSI** offre opzioni di test center e di esame giapponese (ove disponibili)
* **[!DNL Examity]** offre un&#39;opzione online remota

Per pianificare, riprogrammare o annullare l&#39;esame, è sufficiente fare clic sul fornitore desiderato e seguire le istruzioni fornite sullo schermo. È così semplice!
